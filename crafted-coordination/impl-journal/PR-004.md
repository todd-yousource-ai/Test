## What

Implement `InMemoryTaskStore` in `src/tasklib/storage/memory.py` providing a complete in-memory CRUD interface for `Task` objects. Update `src/tasklib/storage/__init__.py` to re-export `InMemoryTaskStore`.

## Why

The task store is the single source of truth for task state within a session (TRD §3.2). It sits between the Task model layer and the CLI layer in the dependency chain. This PR satisfies FR-8 through FR-14 and enables downstream CLI work.

## How

`InMemoryTaskStore` uses a plain `dict[str, Task]` keyed by `task.id`. Five public methods provide the full interface:

- `add(task)` — stores a Task, raises `ValueError` if the ID already exists
- `get(task_id)` — returns a Task, raises `KeyError` if not found
- `list_all()` — returns a list of all tasks (insertion order)
