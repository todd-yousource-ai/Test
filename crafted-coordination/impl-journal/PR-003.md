## What

Implement the `Task` dataclass and `TaskStatus` enumeration in `src/tasklib/models/task.py`, and update `src/tasklib/models/__init__.py` to re-export both symbols.

## Why

The Task model is the foundational data structure for the entire tasklib library. Every downstream component — the in-memory store, the CLI, and the package root re-exports — depends on `Task` and `TaskStatus` existing with a stable interface. This PR satisfies TRD-TASKLIB §3.1 (Task Model) requirements:

- A Task can be created with only a title; `id`, `status`, and `created_at` are populated automatically.
- `TaskStatus` enumeration prevents construction with invalid status values.
- A task serialized to a dictionary and then deserialized produces an equivalent task.
