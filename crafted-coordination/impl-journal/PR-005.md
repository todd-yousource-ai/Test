## What

Create the test package (`tests/__init__.py`) and a test module (`tests/test_scaffold.py`) that verify the scaffold laid down in PR #2 is importable and side-effect-free.

## Why

PRD-002 explicitly requires a test file that "verifies that importing tasklib, tasklib.models, and tasklib.storage succeeds without error or side effects." This PR satisfies that validation gate and ensures all placeholder modules (`task.py`, `store.py`, `cli.py`) are also importable. Additionally, it guards against accidental public-name leakage from scaffold modules that should contain only docstrings, `__all__`, or `pass`.

These checks protect the build pipeline by catching:
- broken package layout
- missing `__init__.py` files
- invalid module paths
- accidental symbol exports from placeholder modul
