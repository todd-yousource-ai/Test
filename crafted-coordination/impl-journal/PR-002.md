## Why

This is PR #2 in the validation dependency chain (`docs → scaffold → model → storage → CLI`). The scaffold establishes the physical package structure that all subsequent PRs depend on. By landing structure separately from logic, we validate that:

1. The merge gate accepts a pure-scaffold PR after the docs-only PR #1.

3. The package is importable as `tasklib` with no runtime errors, even before business logic exists.
